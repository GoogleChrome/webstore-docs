Monetizing Your Products

You have many choices when it comes to making money from products that you distribute on Chrome.

Making Money

To help meet your business needs, Chrome supports a variety of monetization models.

Free with ads

Ads are a great option for monetizing your product without charging your users. For Chrome-integrated websites, consider using Google AdSense.

Freemium

If you charge for your product, we encourage you to provide a free trial version, sometimes called freemium. A free trial should provide a basic level of functionality that lets the user gauge the usefulness of the product. Time-limited trials can prompt users for one-time payment after expiration. Feature-limited trials can offer additional features using in-app payments.

Note: If you list your product as free, it must perform its function without asking for payment from the user. See the policy document for details.

One-time payments

Users buy your product by providing a one-time payment upfront. For more information, see one-time payments.

Note: This is the only monetization option for themes.

Subscription

Users purchase access to your product on a subscription basis. The Chrome Web Store payment solutions support both monthly and yearly subscription models.

Custom

The options above are just a few suggestions for monetization. You are also free to implement a custom model that works best for you. For in-app payments, consider using Chrome Web Store API.

Choosing a payment system

If you’re distributing your product through the Chrome Web Store, you can use Chrome Web Store Payments, a built-in payment system powered by Google Wallet for Digital Goods. Even if you don’t distribute through the Chrome Web Store, you can still use the Chrome Web Store API to manage in-app payments.

Note: Chrome Web Store Payments are not available for Chrome-integrated websites (‘hosted apps’).

Note: Chrome merchants who currently manage their own digital inventories and use Google Wallet for Digital Goods API need to migrate to the Chrome Web Store API or another payment solution before March 2, 2015.

Using a Chrome Web Store payment solution is recommended, because:

  • It provides an integrated experience for users.
  • It is competitively priced for merchants (5% + a small per transaction fee). See Payments: Regions, Fees, and Tiers for details.
  • You can use the Chrome Web Store Licensing API to check on user payments.

Alternatively, you can choose to use any other payment service provider that you prefer. However, these payments will not be integrated with the Chrome Web Store experience.

Best Practices

From first-time users to loyal fans, these best practices can help you earn more revenue by providing great user experiences across the board.

  • Don't put any roadblocks in front of users. Freshly installed products should always provide something useful or interesting, even if the user hasn't paid yet. If the first page the user sees is useless—nothing but a payment or login wall, for example—the user is likely to uninstall your product, and you might get some scathing reviews.
  • Make signup and purchasing easy. If you do require login or payment at some point, use trusted third party services, such as Google Accounts and Chrome Web Payments. These services are convenient, secure, and save your users’ the trouble of remembering yet another username and password.
  • Update often. Increase user engagement by keeping your content fresh and actively maintaining your product. Chrome's support for auto-update makes distributing updates easy.

For more best practices, see the Key Principles of monetization on Web Fundamentals.
